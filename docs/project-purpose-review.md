# TreeCS — 프로젝트 목적 및 구현 검토

> 문서 목적: 시스템의 지향점을 "예측 기반 수목 하자 관리"로 명확히 하고,
> 현재 구현이 그 목적에 부합하는지 점검하여 문구·로직 개선점을 정리한다.

## 1. 프로젝트 목적

### 비전
최근 3년간 축적된 수목 하자 데이터를 기반으로 하자 발생 요인을 분석하고,
설계 단계에서 고위험 조건을 사전에 식별·반영하여 하자를 예방하고
하자 처리 원가를 절감하는 **예측 기반 수목 하자 관리 시스템**.

### 핵심 목표
1. 패턴 분석 — 수종·계절·지역·현장별 하자 발생 패턴 파악
2. 요인 정량화 — 회귀분석으로 각 요인이 하자율에 미치는 영향도 산출
3. 사전 예측 — 설계 단계에서 고위험 조건(예: 특정 수종 겨울 식재)을 위험 요인으로 표시
4. 대체 제안 — 고위험 수종에 대한 대체 수종 추천
5. 원가 절감 — 하자율 저감 → 하자 처리비·예비비 절감으로 연결

### 차별점
단순 데이터 조회 대시보드가 아니라
**분석 → 예측 → 설계 반영 → 원가 절감**으로 이어지는 의사결정 지원 시스템.

### 데이터 한계 명시
데이터 기간이 3년으로 제한적이다. 따라서 결과는 "확정된 인과"가 아니라
"데이터 기반 위험 경향"으로 제시하며, 표본이 적은 조건은 신뢰구간·표본 수를
함께 표기해 과대해석을 방지한다.

## 2. 분석 방법론

### 종속변수(Y)
하자 여부(고사/생존). 현재 데이터는 개체(나무 1주) 단위가 아니라
점검 묶음 단위(`inspection_items`: 점검수량/하자수량)이므로,
**그룹 이항 로지스틱 회귀**(N주 중 D주 하자)로 수행한다.
개체별 고사 시점이 없어 개체 단위 생존분석(Cox)은 불가하나,
한 식재건이 여러 점검회차에 걸쳐 있으면 그룹 단위 이산시간 생존분석은 가능하다.

### 방법 조합 (단일 기법 아님 — 역할 분담)
- **로지스틱 회귀** — 각 팩터의 영향도(odds ratio) 정량화. 보고의 뼈대.
- **혼합효과 모형** — 협력사를 변량효과로 두어 "업체 고유 효과"와 "맡은 현장 조건"을 분리.
- **의사결정나무** — "남부+겨울+점토질" 같은 고위험 조건 조합을 if-then 규칙으로 추출.
- **성향점수 매칭** — 협력사 비교의 공정성 확보(조건 보정 하자율).
- **(데이터 충분 시) XGBoost + SHAP** — 변수 중요도·개별 현장 설명.

> 3년 데이터에서 실제 가능한 모델 범위는 **총 하자 이벤트 수**가 결정한다
> (이벤트 약 10건당 독립변수 1개 원칙). 모델 설계 전 표본 진단이 선행돼야 한다.

## 3. 팩터 체계

| 구분 | 성격 | 주요 팩터 | 출처 | 현 상태 |
|---|---|---|---|---|
| 내부 확정 | 팩트 | 수종, 규격(H/R/근원경), 수량, 협력사, 현장, 지역, 식재일·계절 | TreeCS DB | 보유 |
| 파생 | 계산 | 식재 적기 이탈, 점검~식재 경과, 묶음 하자율 | 내부 계산 | 일부 |
| 외부 추정 | API | 기상(강수·기온·폭염), 토양(배수·토심·토성), 지형(고도·경사) | 기상청/농진청/브이월드 | 미연동 |
| 실측 보완 | 수기 | 산지·반입형태(농장목/산채목, B&B), 객토·배수층, 관수·방제 | 현장 입력 | 미수집 |

> **핵심 병목**: 외부 API 연동의 전제인 현장 좌표(위경도)가 DB에 없다.
> `sites`에는 주소·지역만 있다. 주소→좌표 지오코딩이 선행돼야
> 기상·토양·지형 팩터 전체가 작동한다.

## 4. 현재 구현 vs 목적 — 갭 검토

### 화면 현황
- **마스터·입력**: sites, contractors, species, plantings, inspections (CRUD) — 충실
- **분석**: analytics, risk-analysis, simulation — 집계·시각화 중심
- **대체수종**: species-finder + 엑셀 업로드 매핑
- **설정·업로드·권한**: 구비

### 핵심 갭 — "예측"이라 부르지만 실제는 집계 + 수동 입력
| 항목 | 목적문 표현 | 실제 구현 |
|---|---|---|
| 리스크 분석 | 예측 기반 위험도 | 하자율 ≥20% 고위험 식 **임계값 색칠** (모델 아님) |
| 예상 하자율 | 시스템 예측값 | **엑셀로 업로드된 입력값** (`expected_defect_rate`) |
| 대체수종 추천 | 데이터 기반 추천 | **사람이 만든 매핑표 업로드** (`species_substitutions`) |
| 분석 단위 | 다요인 결합 분석 | 수종별·계절별·지역별 **단변량 집계** (`agg_metrics_by_*`) |

> 즉 목적문의 "회귀분석·예측"은 아직 미구현이며, 현재는 "정리·표시" 단계다.
> 이 격차를 문구와 로직 양쪽에서 메워야 한다.

## 5. 개선 권고

### 문구 — 현 단계를 정직하게 표기 (신뢰 확보)
- "AI 예측" / "하자 예측" → 현재 화면은 **"하자율 현황·위험도 표시"**로 표기.
  회귀 모델 탑재 후 "예측"으로 승격한다.
- "대체수종 추천(AI)" → 현재는 **"권장 대체수종(기준표)"**. 데이터 산출 전까지 AI 표현 보류.
- risk-analysis의 "예상 하자 관리비용"은 **입력 가정 기반**임을 각주로 명시.

### 로직 — 목적 달성 경로
1. **표본 진단** — 총 식재·점검·하자 주수, 식재일 NULL율, 규격 파싱률 확인 (모델 가능 범위 결정)
2. **분석용 flat 뷰 생성** — inspection_items + plantings + sites + species + spec_codes 조인.
   현재 agg_* 단변량 테이블로는 회귀 불가.
3. **좌표 지오코딩** — `sites.address` → lat/lng 컬럼 추가. 외부 API 연동의 전제.
4. **회귀 파이프라인** — 로지스틱 → 혼합효과 → 트리 순. 결과(odds ratio, 조건보정 협력사 효과)를 DB 적재.
5. **expected_defect_rate 산출 전환** — 수동 입력 → 회귀 모델 출력으로 교체.
   이때 비로소 화면에 "예측"으로 표기 가능.
6. **대체수종 로직 고도화** — 수동 매핑 → 동일 조건(지역·계절·토양) 대비 저하자율 수종 자동 추천.

### 로드맵
- **1단계 (현재 가능)**: 표본 진단 + flat 뷰 + 문구 정정
- **2단계**: 좌표 지오코딩 + 외부 기상·토양 API 연동
- **3단계**: 회귀 모델 탑재 → expected_defect_rate 자동화 → "예측" 승격
- **4단계**: 대체수종 자동 추천 + 설계 단계 위험 경고
